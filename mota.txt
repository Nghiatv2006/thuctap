12	Description	Mục tiêu
- Nắm cơ bản quy trình phát triển phần mềm (phân tích, thiết kế, lập trình, kiểm thử).
- Hiểu nghiệp vụ quản lý kho: quản lý người dùng, sản phẩm, nhập/xuất kho, tồn kho.
- Rèn kỹ năng lập trình Java, kết nối MySQL/PostgreSQL, và xây dựng giao diện JavaFX.
- Phát triển kỹ năng làm việc nhóm theo quy trình Agile/Scrum cơ bản.
Hình thức thực hiện
- Tổ chức lớp học có hướng dẫn:
- Kết hợp lý thuyết (30%) và thực hành (70%), sử dụng phương pháp "Tích hợp (Hướng dẫn + Tự thực hành)".
- Giảng viên cung cấp tài liệu (Syllabus, Slide, Template) và mã nguồn mẫu.
- Hỗ trợ qua email, nhóm chat (Zalo/Teams), hoặc Q&A trực tuyến.
- Rèn luyện kỹ năng phân tích và thiết kế:
- Sinh viên vẽ Use Case, ERD, Wireframe đơn giản.
- Sinh viên yếu: Mô tả Use Case hoặc vẽ Wireframe cơ bản.
- Sinh viên trung bình: Vẽ ERD hoặc Activity Diagram.
- Sinh viên giỏi: Chuẩn hóa ERD hoặc tối ưu thiết kế.
- Rèn luyện kỹ năng lập trình:
- Sinh viên lập trình module đăng nhập, quản lý sản phẩm, nhập/xuất kho, báo cáo cơ bản.
- Sinh viên yếu: Code giao diện hoặc CRUD đơn giản.
- Sinh viên trung bình: Code logic nhập/xuất kho.
- Sinh viên giỏi: Tối ưu truy vấn SQL hoặc giao diện.
- Sinh viên làm việc nhóm:
- Nhóm 3–5 sinh viên với vai trò (Scrum Master, Developer, Tester, Documenter).
- Quy trình Agile/Scrum:
- Sprint: 1 tuần, cập nhật Backlog trên Trello.
- Daily Scrum: Họp 10 phút/ngày.
- Sprint Review: Đánh giá với giảng viên.
- Phân vai trò:
- Sinh viên yếu: Documenter hoặc Tester.
- Sinh viên trung bình: Developer (code cơ bản).
- Sinh viên giỏi: Scrum Master hoặc Developer (logic nâng cao).
- Đánh giá: Nộp Worklog cá nhân ghi công việc và thời gian.

7. Công cụ bổ trợ khác
